# SecureX 일일 업무보고서 자동 생성기

SecureX 기술지원팀의 일일 업무보고서 자동 생성 앱 실습 프로젝트입니다.

메신저 대화 내용과 JIRA CSV 데이터를 입력받아 AI 기반 일일 업무보고서를 자동으로 생성합니다.
OpenAI API를 통한 보고서 생성과 API Key 없는 로컬 샘플 생성 두 가지 방식을 지원합니다.

## 주요 기능

- ✅ **업무 현황 요약표**: 메신저와 JIRA 데이터를 종합한 HTML 테이블 형식
- ✅ **AI 보고서 생성**: OpenAI API를 통한 자동 작성
- ✅ **로컬 샘플 보고서**: API Key 없이 템플릿 기반 생성
- ✅ **보고서 복사**: 요약표 + 본문을 마크다운 형식으로 클립보드에 복사
- ✅ **MD 다운로드**: `securex_daily_report_YYYY-MM-DD.md` 마크다운 파일로 저장
- ✅ **한국어 오류 메시지**: 사용자 친화적인 안내 메시지
- ✅ **다크 모드 UI**: 보안 회사 업무 도구 느낌의 디자인

## 파일 구성

- `index.html`: 웹앱의 화면 구조
- `style.css`: 다크 모드 스타일 및 테이블 디자인
- `app.js`: 전체 기능 로직 (API 연결, 데이터 처리, 오류 처리)
- `messenger_chat_sample.txt`: 메신저 대화 샘플 데이터
- `jira_task_sample.csv`: JIRA 업무 샘플 데이터
- `README.md`: 이 파일

## 실행 방법

샘플 데이터 불러오기는 브라우저 보안 정책상 로컬 웹 서버에서 실행해야 합니다.

### 1단계: 웹 서버 실행

현재 폴더에서 PowerShell/Terminal을 열고 다음 중 하나를 실행합니다.

**Windows (Python 설치됨)**
```powershell
python -m http.server 8000
```
또는
```powershell
py -m http.server 8000
```

**macOS / Linux**
```bash
python3 -m http.server 8000
```

### 2단계: 브라우저에서 접속

```
http://localhost:8000
```

### 3단계: 앱 사용

아래 "최종 테스트 순서" 섹션을 참고하여 기능을 테스트합니다.

## 오류 처리 및 안내 메시지

앱은 다음과 같은 상황을 한국어로 안내합니다:

| 상황 | 메시지 |
|------|--------|
| 샘플 파일 없음 | ⚠ 샘플 파일을 불러오지 못했습니다. 로컬 웹 서버로 실행해 주세요. |
| 입력 데이터 없음 | ⚠ 메신저 또는 JIRA 데이터를 입력해 주세요. |
| API Key 미설정 | ⚠ API Key가 설정되지 않았습니다. |
| API 인증 오류 | API Key가 유효하지 않습니다. |
| API 속도 제한 | API 호출 한도를 초과했습니다. 잠시 후 다시 시도해 주세요. |
| API 서버 오류 | OpenAI 서버 오류입니다. 잠시 후 다시 시도해 주세요. |
| 클립보드 권한 없음 | ⚠ 클립보드 복사 권한이 없습니다. 마우스로 직접 선택 후 복사해 주세요. |

## 기술 스택

- **Frontend**: HTML5, CSS3, Vanilla JavaScript
- **API**: OpenAI Chat Completions API
- **Storage**: Browser LocalStorage (API Key)
- **Data Format**: CSV (JIRA), Text (Messenger)
- **Encoding**: UTF-8 / EUC-KR 자동 감지

## 보안 정책

- ✅ API Key는 LocalStorage에만 저장 (서버 전송 안 함)
- ✅ API Key는 코드에 하드코딩 안 됨
- ✅ 민감정보 마스킹 안내
- ✅ 개발자 도구에 API Key 원문 출력 안 함
- ⚠️ **주의**: LocalStorage는 실습용 임시 저장소입니다. 실제 운영 환경에서는 서버 기반 API Key 관리 필요

## 문제 해결

| 문제 | 해결 방법 |
|------|---------|
| "샘플 파일을 불러오지 못했습니다" | 로컬 웹 서버 실행 확인: `python -m http.server 8000` |
